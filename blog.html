Questions1 : What's the difference : map(), forEach(), filter(),find() methods?

Answer :.map() executes the same code on every element in an array and returns a new array with the updated elements.


.forEach(), is used to execute the same code on every element in an array but does not change the array and it returns undefined.


.filter() checks every element in an array to see if it meets a certain criteria and returns a new array with the elements that return truthy for the criteria.


.find() is also a search function like the previous but they differ in one small detail — this function returns only one match in an array. If in an array is more than one result, the function will return the first that has matched.

Questions2 :What's the difference : local Storage, session storage, Cookies?


Answer : Local Storage : It is a web storage method that helps us store data on the client’s computer in the form of key/value pairs in a web browser. The data is stored in local storage for a lifetime unless the user manually deletes it from the browser. It does not expire even when the user closes the window or tab. Instead, the data remains in the browser until and unless the browser's memory is cleared.


Session Storage : It is very similar to the local storage. Still, the main difference lies in the lifespan as it persists in the browser until its current tab is on. Once you close the tab or terminate it, the data on session storage also gets lost. We can also set and retrieve its data using setItem() and getItem() methods, respectively, similar to the local storage methods.


Cookies : An HTTP cookie is a piece of data stored by the user's web browser. Unless otherwise specified, cookies can be created, read, modified and deleted both by JavaScript and from server-side through HTTP headers.


Questions3 :What is Event loop ? how does it work ?

Answer :The event loop is a key component of JavaScript's concurrency model that manages the execution of asynchronous code.During the execution of a task, the event loop temporarily halts the execution of other code, but it continues to monitor the task queue for new tasks. Once the current task is completed, the event loop resumes the execution of any other pending tasks in the task queue.
